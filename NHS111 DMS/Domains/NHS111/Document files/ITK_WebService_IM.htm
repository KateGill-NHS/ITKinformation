1    Application Roles

The applications involved in the NHS 111 processes play specific roles.  These, along with the interactions associated with each role, are identified below.

1.1    NHS 111 Originating System - POCD_AR000004GB01

The NHS 111 Originating System is responsible for sending the following NHS 111 interactions:
Primary General Practitioner Recipient NHS 111 CDA Document
Primary Emergency Department Recipient NHS 111 CDA Document
Primary Out of Hours Recipient NHS 111 CDA Document
Primary Repeat Caller Repository Recipient NHS 111 CDA Document
Copy NHS 111 CDA Document
Nullify NHS 111 CDA Document
Primary NHS 111 Request for Ambulance
NHS 111 Repeat Caller Sync Query Response
Primary NHS 111 Repeat Caller Document Set
The NHS 111 Originating System may receive responses to the interactions from the Receiving System.
NHS 111 Repeat Caller Sync Query Response

Note:- the NHS 111 Originating System may not necessarily send all the interactions detailed above but when sending any interaction SHOULD obey the following rules:

 

Message flow Rule
When sending a NHS 111 CDA Document
SHOULD issue a 'Primary Recipient NHS 111 CDA Document' to a recipient (or recipients) where the report contains an implied action for that recipient, with said recipient identified as a primary recipient.
MAY issue a 'Copy NHS 111 CDA Document' to a recipient where the report is for information or does not contain an implied action. The recipient should be a copy-to (tracker) recipient in this case

1.2    NHS 111 General Practitioner Receiving System - POCD_AR000005GB01

The NHS 111 General Practitioner Receiving System receives and should indicate to the user that recommendations are contained in the document of the following NHS 111 interactions when said user is identified as a General Practitioner recipient of the document:
Primary General Practitioner Recipient NHS 111 CDA Document
The NHS 111 General Practitioner Receiving System receives and SHOULD act on the following NHS 111 interaction:
Nullify NHS 111 CDA Document
The NHS 111 General Practitioner Receiving System may initiate responses for the interaction to the NHS 111 Originating System.

1.3    NHS 111 Emergency Department Receiving System - POCD_AR000006GB01

The NHS 111 Emergency Department Receiving System receives and should indicate to the user that recommendations are contained in the document of the following NHS 111 interactions when said user is identified as a Emergency Department recipient of the document:

1.5    NHS 111 Repeat Caller Repository Receiving System - POCD_AR000008GB01

The NHS 111 Repeat Caller Repository Receiving System receives and should indicate to the user that recommendations are contained in the document of the following NHS 111 interactions when said user is identified as a Repeat Caller Repository recipient of the document:

1.6    NHS 111 Copy Recipient - POCD_AR000009GB01

The NHS 111 Copy Recipient receives the following NHS 111 interaction:
Copy NHS 111 CDA Document
The NHS 111 Copy Recipient receives and SHOULD act on the following NHS 111 interaction:
Nullify NHS 111 CDA Document
The NHS 111 Copy Recipient may initiate responses for the above interactions to the NHS 111 Originating System.

1.7    NHS 111 Ambulance Request Receiving System - REPC_AR000001GB01

The NHS 111 Ambulance Request Receiving System receives the Ambulance Request from NHS 111 Originating System and should take action to cater to request message:
Primary NHS 111 Request for Ambulance
The NHS 111 Ambulance Request Receiving System may initiate responses for the interaction to the NHS 111 Originating System.

1.8    NHS 111 Repeat Caller Query Fulfiller System - QUPA_AR000001GB01

The NHS 111 Repeat Caller Query Receiving System receives the Query Request from NHS 111 Originating System and should take action to cater to query message:
NHS 111 Repeat Caller Query
The NHS 111 Repeat Caller Query Receiving System may initiate responses for the interaction to the NHS 111 Originating System.

 

2    Trigger Events

2.1    NHS 111 Send - POCD_TE000003GB01

The NHS 111 Send trigger event indicates that a NHS 111 CDA document is ready to send from the originating system.

2.2    NHS 111 Nullify By Replacement - POCD_TE000004GB01

For the NHS 111 CDA Document to be nullified the trigger event is the realisation that the NHS 111 CDA Document was sent in error, in that it contained an error and should never have been sent and that no correct NHS 111 CDA Document is available to send.

2.3    NHS 111 Ambulance Request - REPC_TE000001GB01

The NHS 111 Ambulance Request trigger event indicates that an Ambulance is requested.

2.4    NHS 111 Repeat Caller Query - QUPA_TE000001GB01

The NHS 111 Repeat Caller Query trigger event indicates that an Query is triggered.

2.5    NHS 111 Repeat Caller Response - PRPA_TE000001GB01

The NHS 111 Repeat Caller Response trigger event indicates that an Response is triggered.

 

 

3    Interactions

Note: NHS CDA documents are bound to the "Balloted CDA model" and use the "NHS CDA Generic schema (POCD_MT000002UK01.xsd)". The CDA Domain models documented in this pack under the "Message Definitions" tab, are used for conformance checking of NPfIT templates after transforming the "On-the-Wire" instance to "templated" instance. See the "Transforms" tab under "References" for details of a transform from On-the-Wire to a templated instance.

 

This specification defines responsibilities additional to the standard CDA recipient and originator responsibilities. These are documented in the Generic CDA Specification. Use of CDA replacement is documented in the Generic CDA Specification. For this CDA document profile these additional responsibilities are option.
