1    Application Roles

The applications involved in the NHS 111 processes play specific roles.  These, along with the interactions associated with each role, are identified below.

1.1    NHS 111 Originating System - POCD_AR000001GB01

The NHS 111 Originating System is responsible for sending the following NHS 111 interactions:
Primary Recipient NHS 111
Copy NHS 111
Nullify NHS 111
The NHS 111 Originating System may receive responses to the interactions from the Receiving System.

Note:- the Non-Coded CDA Document Originating System may not necessarily send all the interactions detailed above but when sending any interaction SHOULD obey the following rules:

 

Message flow Rule
When sending a NHS 111
SHOULD issue a 'Primary Recipient NHS 111' to a recipient (or recipients) where the report contains an implied action for that recipient, with said recipient identified as a primary recipient.
MAY issue a 'Copy NHS 111' to a recipient where the report is for information or does not contain an implied action. The recipient should be a copy-to (tracker) recipient in this case

1.2    NHS 111 Primary Receiving System - POCD_AR000002GB01

The NHS 111 Primary Receiving System receives and should indicate to the user that recommendations are contained in the document of the following NHS 111 interactions when said user is identified as a primary recipient of the document:
Primary NHS 111
The NHS 111 Primary Receiving System receives and SHOULD act on the following NHS 111 interaction:
Nullify NHS 111
The NHS 111 Primary Receiving System may initiate responses for the interaction to the NHS 111 Originating System.

1.3    NHS 111 Copy Recipient - POCD_AR000003GB01

The NHS 111 Copy Recipient receives the following NHS 111 interaction:
Copy NHS 111
The NHS 111 Copy Recipient receives and SHOULD act on the following Non-Coded CDA Document interaction:
Nullify NHS 111
The NHS 111 Copy Recipient initiates responses for the above interactions to the NHS 111 Originating System.

2    Trigger Events

2.1    NHS 111 Send - POCD_TE000001GB01

The NHS 111 Send trigger event indicates that a NHS 111 is ready to send from the originating system.

2.2    NHS 111 Nullify By Replacement - POCD_TE000002GB01

For the Non-Coded CDA Document to be nullified the trigger event is the realisation that the NHS 111 was sent in error, in that it contained an error and should never have been sent and that no correct NHS 111 is available to send.

4    Interactions

Note:- For CDA domains when using ITK the interactions are bound to the "Balloted CDA model" and use the "NHS CDA Generic schema (POCD_MT000002UK01.xsd)"
The CDA Domain models are used for conformance checking of NPfIT templates after transforming the "On the Wire" instance to "templated" instance.

This specification defines responsibilities additional to the standard CDA recipient and originator responsibilities. These are documented in the Generic CDA Specification. Use of CDA replacement is documented in the Generic CDA Specification. For this CDA document profile these additional responsibilities are option.
