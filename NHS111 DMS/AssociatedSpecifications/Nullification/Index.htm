Overview: Nullification Domain Message Specification

The Nullification Domain Message Specification supports the following care communications:

  • Nullification CDA Document

This domain details generic documents used to nullify documents sent by CDA domains. Nullification documents are used where it is not possible to use the normal replacement mechanism because no correcting document is available. Further details of the normal replacement mechanism are contained within each CDA domain.


Domain Compliance Statement:  The CDA document specified within this domain complies with the conformance statement from "HL7 Clinical Document Architecture, Release 2.0" (2006 Normative Edition), specifically:


"A conformant CDA document is one that at a minimum validates against the CDA Schema, and that restricts its use of coded vocabulary to values allowable within the specified vocabulary domains." The CDA specification described in this domain makes use of a templating mechanism that supports both entry-level and section-level templating. NHS CFH have implemented the HL7 guidelines for CDA and have included localisations to aid navigation of the CDA document.


Use of templates: This domain uses templates, for further information on templates refer to the latest version of "NPFIT-ELIBR-AREL-P1R2-0178 Technical Guidance for Implementation of Templated CDA Domains"
